HuntVoidWatcher Release Notes

## 0.4.8.76 - 2026-03-15
- FATE通知の消滅/完了メッセージ更新時も、地図画像をEmbed内表示のまま維持するよう修正。
- 消滅/完了時のWebhookメッセージ編集で、画像URLとattachments情報を保持して再送する処理を追加。
- これにより、更新後に画像がEmbed外へ落ちる不具合を修正。
運用ルール:
- バージョン更新時は必ず ReleaseNote.txt を更新する。
- 配布用リポジトリにはソースコードを含めない。

## 0.4.8.75 - 2026-03-15
- FATE地図通知ON時の重複・表示崩れ対策を強化。
- spawn送信中（MessageId未確定）の `progress_tick` 送信を抑止し、初回送信とのレースを回避。
- 添付付き送信後に添付URLが取得できなかった場合、メッセージ再取得で添付URLを補完。
- 進捗更新時もEmbed image URLを維持し、画像がEmbed外表示へ崩れるケースを軽減。

## 0.4.8.74 - 2026-03-15
- FATE通知Embedの文面を簡潔化（サーバー名/FATE名/日付/時刻の項目名を削除）。
- 「進捗率」表記を「進行度」に統一。
- 状態「進捗」を細分化:
  - 進行度0%: 発生中
  - 進行度1%以上: 進行中
- 地図通知ON時のEmbed画像維持を改善:
  - ファイル送信レスポンスから添付URLを保持し、後続のメッセージ編集時もEmbed imageに再指定。

## 0.4.8.73 - 2026-03-15
- FATE名が出現直後に `「」` になるケースを軽減。
- spawn送信前に短時間でFATE名を再サンプリングし、名前が安定してから通知本文・送信者名を作成。
- Discordの送信者名が誤った空名のまま固定されるケースを抑制。

## 0.4.8.72 - 2026-03-15
- FATE通知Embedから「地図添付: ON」表示を削除（不要表示の整理）。
- FATE状態監視周期を 500ms から 50ms に短縮。
- 進捗が90%台で止まるケースの早期追従を狙って監視解像度を向上。

## 0.4.8.71 - 2026-03-15
- FATE地図マーキング座標の安定化処理を追加。
- 出現直後に座標を短時間再サンプリング（約0.9秒 + 0.45秒）してから地図を生成。
- 同一エリアでも発生していた「ときどき大きくズレる」ケースの抑制を狙った改善。

## 0.4.8.70 - 2026-03-14
- 一部FATEで `territoryId=0` になり地図生成が `No map source (0)` となる問題を改善。
- FATE行からエリアID解決に失敗した場合、現在のクライアントエリアIDへフォールバックする処理を追加。

## 0.4.8.69 - 2026-03-14
- Discordファイル送信時の `attachments` メタデータを `payload_json` に明示。
- FATE地図通知で `attachment://...` 参照が解決されず、Embed内画像が表示されないケースを改善。

## 0.4.8.68 - 2026-03-14
- FATE通知の「地図添付」表示を調整。
- 地図通知設定がOFFの場合は「地図添付: OFF」を表示しない（項目自体を非表示）。
- 地図通知設定がONの場合のみ「地図添付: ON」を表示。

## 0.4.8.67 - 2026-03-14
- FATE 100%終了時の状態表示を修正。
- 100%で終了した場合、状態は「消滅」ではなく「完了」で表示。
- 100%終了時のEmbedカラーを緑に変更（消滅時は従来どおり赤）。

## 0.4.8.66 - 2026-03-14
- FATE通知の報告許可判定を安定化。
- メインスレッド外で `ObjectTable.LocalPlayer` を参照していた処理を修正し、`Not on main thread` 例外を回避。
- spawn送信に失敗した場合でも、最初の `progress_tick` 送信前に報告許可API判定を行うよう改善。
- これにより、spawn経路の例外で許可判定が抜けるケースを抑制。

## 0.4.8.65 - 2026-03-14
- FATEの地図通知表示を修正。
- 地図通知設定がONのFATEでは、進捗更新後もEmbed本文の「地図添付」をON表示に統一。
- FATE 100%完了時の残り時間表示を修正。
- 消滅通知で残り時間を `00:00` 固定にせず、100%到達時点（または直前有効値）の残り時間を保持して表示。

## 0.4.8.64 - 2026-03-14
- FATE地図通知ON時のEmbed表示を修正。
- 地図添付が成功した場合、Embed本文の「地図添付」を `ON` のまま維持。
- 添付画像をEmbed内に表示するため、`attachment://...` 参照を設定。
- 進捗更新でメッセージ編集された後も、地図添付状態が `OFF` に誤上書きされないよう修正。

## 0.4.8.63 - 2026-03-14
- 通知輻輳防止のため、Apps Script Web API に「報告許可」を問い合わせる機能を追加。
- API応答が正常で `allow=false` の場合は通知をスキップ（正常系としてフォールバックしない）。
- API URL未設定・異常応答・アクセス失敗時はフォールバックとして通常どおり通知を送信。
- 設定画面を `App Script1` / `App Script2` に分離。
- `App Script1`: スプレッドシート在席同期用のスクリプトとURL設定。
- `App Script2`: 通知許可判定（輻輳防止）用のスクリプトとURL設定。

## 0.4.8.62 - 2026-03-14
- FATE通知から「近傍プレイヤー画像」の添付を廃止。
- 近傍プレイヤー情報は地図通知ON/OFFに関係なくEmbed本文へ統合（進捗1%以上で更新）。
- 地図通知ON時はFATE発生時にPOP位置をエリア地図へマーキングした画像を添付（Sモブ通知と同系ロジック）。
- Embed本文の表示順序を調整:
  サーバー名 / FATE名 / 残り時間 / 進捗率 / 状態 / 日付 / 時刻 / 近傍プレイヤー / 地図添付。
- エリア名行はEmbed本文から削除（送信者名に含まれるため）。

## 0.4.8.61 - 2026-03-14
- FATE状態監視ポーリングは 500ms のまま維持。
- 定期メッセージ更新は 5秒間隔（`progress_tick`）を維持。
- `progress_100` は5秒制約を無視して即時更新するよう修正。
- `spawn` / `despawn` は従来どおり即時更新。

## 0.4.8.60 - 2026-03-14
- MainUI（`HVW /sea` 小型UI）のドラッグ移動判定を修正。
- これまで移動を阻害していた `IsAnyItemActive()` ガードを外し、ウィンドウ上で左ドラッグ時に確実に移動するよう改善。

## 0.4.8.59 - 2026-03-14
- FATE消滅時の最終進捗補正を追加。
- 消滅直前サンプルが 98〜99% で止まった場合でも、完了扱いで最終進捗を 100% として通知。
- これにより「実際は完了しているのに最終進捗が98%表示で止まる」ケースを軽減。

## 0.4.8.57 - 2026-03-14
- FATE進捗通知をマイルストーン方式から定期更新方式へ変更。
